Shared Preferences

¿Qué es SharedPreferences?​

SharedPreferences es una herramienta proporcionada por Android para almacenar datos simples en forma de pares clave-valor. Es ideal para guardar configuraciones, preferencias del usuario y datos pequeños que deben persistir entre sesiones.

Casos de Uso​

  • Guardar configuraciones (modo oscuro, idioma preferido, etc.).
  • Persistir pequeños datos, como el estado de inicio de sesión.
  • Almacenar valores de tipo String, int, float, boolean o Set<String>.

Características​

  1. Persistencia: Los datos permanecen almacenados incluso si la app se cierra o el dispositivo se reinicia.
  2. Privacidad: Los datos son privados a la aplicación.
  3. Facilidad de uso: Proporciona métodos simples para guardar y recuperar valores.

Tips para Usar SharedPreferences​

  1. Organiza las claves: Usa constantes para definir las claves y evitar errores tipográficos.
  2. Evita almacenar datos grandes: SharedPreferences no está diseñado para manejar datos complejos o voluminosos.
  3. Realiza operaciones de lectura en el hilo principal: Son rápidas y no afectan el rendimiento.
  4. Usa apply() en lugar de commit() para evitar bloqueos en el hilo principal.
  5. Considera la seguridad: No uses SharedPreferences para almacenar datos sensibles como contraseñas (usa cifrado o almacenamiento seguro).

Código de Ejemplo​

1. Guardar datos en SharedPreferences​

// Crear o acceder a un archivo de preferencias
SharedPreferences sharedPreferences = getSharedPreferences("MisPreferencias", Context.MODE_PRIVATE);

// Editor para modificar los valores
SharedPreferences.Editor editor = sharedPreferences.edit();

// Guardar valores
editor.putString("usuario", "Juan");
editor.putInt("edad", 25);
editor.putBoolean("notificaciones_activadas", true);

// Aplicar los cambios (asíncrono)
editor.apply(); // Usa commit() si necesitas confirmación inmediata

2. Leer datos de SharedPreferences​

// Acceder al archivo de preferencias
SharedPreferences sharedPreferences = getSharedPreferences("MisPreferencias", Context.MODE_PRIVATE);

// Recuperar valores
String usuario = sharedPreferences.getString("usuario", "Usuario no registrado");
int edad = sharedPreferences.getInt("edad", 0);
boolean notificaciones = sharedPreferences.getBoolean("notificaciones_activadas", false);

// Mostrar los datos
Log.d("SharedPreferences", "Usuario: " + usuario + ", Edad: " + edad + ", Notificaciones: " + notificaciones);

3. Eliminar un dato​

// Eliminar una clave específica
SharedPreferences.Editor editor = sharedPreferences.edit();
editor.remove("edad");
editor.apply();

4. Limpiar todos los datos​

// Limpiar todas las claves y valores
SharedPreferences.Editor editor = sharedPreferences.edit();
editor.clear();
editor.apply();

Errores Comunes​

  1. Olvidar llamar a apply() o commit(): Los cambios no se guardan hasta que se llama explícitamente a estos métodos.
  2. Acceso simultáneo desde múltiples hilos: Puede causar inconsistencias. Evítalo en escenarios multihilo complejos.
  3. Almacenar datos grandes o sensibles: Esto puede afectar el rendimiento y la seguridad.

Buenas Prácticas​

  1. Usar constantes para las claves:

    public static final String PREFS_NAME = "MisPreferencias";
    public static final String KEY_USUARIO = "usuario";
    public static final String KEY_EDAD = "edad";

    Luego:

    editor.putString(KEY_USUARIO, "Juan");

  2. Encapsular lógica en una clase helper:

    public class SharedPreferencesHelper {
        private static final String PREFS_NAME = "MisPreferencias";
        private SharedPreferences sharedPreferences;

        public SharedPreferencesHelper(Context context) {
            sharedPreferences = context.getSharedPreferences(PREFS_NAME, Context.MODE_PRIVATE);
        }

        public void guardarUsuario(String usuario) {
            sharedPreferences.edit().putString("usuario", usuario).apply();
        }

        public String obtenerUsuario() {
            return sharedPreferences.getString("usuario", "Usuario no registrado");
        }

        public void limpiarTodo() {
            sharedPreferences.edit().clear().apply();
        }
    }

    Uso:

    SharedPreferencesHelper prefsHelper = new SharedPreferencesHelper(context);
    prefsHelper.guardarUsuario("Juan");
    String usuario = prefsHelper.obtenerUsuario();

Conclusión​

SharedPreferences es una herramienta poderosa para persistir datos simples en Android. Siguiendo las buenas prácticas y comprendiendo sus limitaciones, se puede aprovechar al máximo esta funcionalidad para mejorar la experiencia del usuario.
