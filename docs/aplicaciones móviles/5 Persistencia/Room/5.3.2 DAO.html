5.3.2 DAO (@Dao)

Un DAO (Data Access Object) es una interfaz o clase abstracta que define cómo interactuar con los datos almacenados en una base de datos Room. Este componente actúa como un puente 🌉 entre las consultas SQL y la aplicación, proporcionando métodos que encapsulan las operaciones de inserción, eliminación, actualización y consulta.

¿Qué hace un DAO? 🤔​

Un DAO permite:

  1. Acceso estructurado a los datos mediante métodos bien definidos 📋✅.
  2. Seguridad y validación en tiempo de compilación 🛡️: Room verifica que las consultas SQL sean válidas y coincidan con el esquema de la base de datos.
  3. Integración con arquitecturas modernas 🚀: Puedes usar LiveData, Flow, o trabajar con coroutines en los métodos DAO.

Cómo crear un DAO 🛠️​

1. Anotar la interfaz o clase abstracta con @Dao​

La anotación @Dao identifica la clase o interfaz como un DAO. 🏷️

@Dao
public interface UserDao {
    ...
}

2. Definir métodos para interactuar con los datos​

Los métodos en un DAO están anotados con operaciones específicas como @Insert ➕, @Update ♻️, @Delete ❌, y @Query 🔍.

Ejemplo básico de un DAO 📘​

A continuación, se muestra un ejemplo de un UserDao que permite realizar operaciones comunes como insertar usuarios, consultar todos los usuarios y buscar un usuario por su ID:

import androidx.room.Dao;
import androidx.room.Insert;
import androidx.room.Query;

import java.util.List;

@Dao
public interface UserDao {

    // Insertar un usuario
    @Insert
    void insertUser(User user);

    // Consultar todos los usuarios
    @Query("SELECT * FROM users")
    List<User> getAllUsers();

    // Consultar un usuario por su ID
    @Query("SELECT * FROM users WHERE id = :userId")
    User getUserById(int userId);
}

Anotaciones de los métodos DAO 📝​

1. @Insert​

  • Inserta una o varias filas en la base de datos.
  • Si el campo anotado con @PrimaryKey ya existe, genera una excepción.
@Insert
void insertUser(User user);

2. @Update​

  • Actualiza una o varias filas basándose en su clave primaria.
@Update
void updateUser(User user);

3. @Delete​

  • Elimina una o varias filas basándose en su clave primaria.
@Delete
void deleteUser(User user);

4. @Query​

  • Ejecuta consultas SQL personalizadas.
  • Es compatible con operaciones SELECT, INSERT, UPDATE y DELETE.
@Query("SELECT * FROM users WHERE first_name LIKE :name")
List<User> findUsersByName(String name);

Tipos de retorno admitidos 🧾​

Room permite diferentes tipos de retorno en los métodos DAO, incluyendo:

  • Listas 🗂️: Para devolver múltiples filas. 
    @Query("SELECT * FROM users")
    List<User> getAllUsers();
  • Objetos individuales 🧑‍💻: Para devolver una sola fila. 
    @Query("SELECT * FROM users WHERE id = :userId")
    User getUserById(int userId);
  • LiveData 📡: Para observar cambios en los datos. 
    @Query("SELECT * FROM users")
    LiveData<List<User>> getAllUsersLiveData();

Relaciones en los DAOs 🔗​

Si tienes tablas relacionadas, puedes usar la anotación @Transaction para manejar consultas que incluyan múltiples tablas. Esto asegura que las operaciones se realicen de forma atómica. 💡

Ejemplo: Relación 1:N​

Imagina que un usuario puede tener múltiples órdenes. La relación se modela así:

Entidad Order:

@Entity(
    foreignKeys = @ForeignKey(
        entity = User.class,
        parentColumns = "id",
        childColumns = "userId",
        onDelete = ForeignKey.CASCADE
    )
)
public class Order {
    @PrimaryKey(autoGenerate = true)
    public int orderId;

    public int userId; // Clave foránea que referencia a "users"

    public String orderDescription;
}

Clase de relación UserWithOrders:

import androidx.room.Embedded;
import androidx.room.Relation;

import java.util.List;

public class UserWithOrders {
    @Embedded
    public User user;

    @Relation(
        parentColumn = "id",
        entityColumn = "userId"
    )
    public List<Order> orders;
}

Método en el DAO:

@Dao
public interface UserDao {

    @Transaction
    @Query("SELECT * FROM users WHERE id = :userId")
    UserWithOrders getUserWithOrders(int userId);
}

Buenas prácticas para DAOs ✅​

sigue estos consejos...
  1. Mantenlo limpio y específico 🧹:
    • Cada DAO debe estar enfocado en una sola entidad o grupo de entidades relacionadas.
  2. Evita operaciones en el hilo principal 🚫🧵:
    • Usa coroutines, LiveData, o Flow para mantener las operaciones fuera del MainThread.
  3. Divide métodos complejos 🧩:
    • Si una consulta SQL es muy larga, divídela en partes lógicas o usa vistas materializadas.
